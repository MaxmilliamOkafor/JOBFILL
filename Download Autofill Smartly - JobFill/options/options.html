  • Custom-fields feature allows you to create your own fields for filling automatically.
  • A custom field has 3 parts including: description, matching condition for labels and filling text/paragraph (or matching condition for select/radio options).
    • Description: simply to describe the purpose of the custom field
    • Matching condition for labels: is a set of one or many conditions used to identify the label
    • Text/Paragraph: will be used to fill in inputs/textareas if their labels are matched with the matching conditions
    • Matching condition for options: similar to the condition for labels, this condition is to select options in Select/Radio fields
  • About matching conditions:
    • Operators:
      • = means Equal
      • ^ means Start With
      • ! means Not
      • && means And
      • || means Or
    • Examples:
      • (name) will match with any label that contains the word name
      • (=name) will match with the label name, but not first name, last name,...
      • (^name) will match with any label that starts with name, example: name please...
      • (!name) will match with any label that does not contain the word name, example: your email:,...
      • (first && name) will match with any label that contains the word first and name, ex: fill in your first name
      • (!(first && name)) will match with any label that does not contain both the words first and name, ex: fill in your last name
      • (position || title) will match with any label that contains the word position or title
      • (=male || (male && !female)) will match with the label male or any label that contains the word male and does not contain female
      • ((expect || desired) && (salary || pay)) will match with any label that contains either the words expect or desired and either the words salary or pay, ex: expected salary, desired pay,..
      • (=expect good salary) will match with the label expect good salary
  • Notes:
    • Always use parentheses outside of the condition, for example: (prof && (asst || assistant || assist)) instead of prof && (asst || assistant || assist)
    • For select/radio fields, the matching condition for options is similar to the condition for labels, it is used to select the options
